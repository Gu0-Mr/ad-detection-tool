# 广告检测工具 - Android Accessibility App

一款基于Android无障碍服务的智能广告检测工具，可以自动检测并提示用户点击广告弹窗的关闭按钮。

## 功能特性

### 核心功能
- 🎯 **智能检测** - 自动检测所有APP右上角的广告关闭按钮
- 📍 **精准区域** - 仅检测屏幕右侧15% × 上方20%区域
- 🔍 **多维识别** - 支持文字(✕、×、X、关闭、跳过)和控件ID识别

### 提醒方式
- 📳 **震动反馈** - 可调节震动强度和时长
- 🔊 **声音提示** - 检测到广告时播放提示音
- 👁️ **悬浮窗** - 实时显示检测状态和计数

### 检测关键词
| 类型 | 关键词 |
|------|--------|
| 符号 | ✕、×、X |
| 中文 | 关闭、关、跳过、不再显示 |
| 英文 | close、skip、dismiss、cancel |
| ID匹配 | close、dismiss、skip、btn_close |

## 快速开始

### 1. 克隆项目
```bash
git clone <repository-url>
cd 广告检测工具_stable
```

### 2. 配置环境
- 安装 Android Studio Hedgehog (2023.1.1) 或更高版本
- 安装 JDK 17
- 确保 Android SDK 34 已安装

### 3. 构建项目
```bash
# 使用Gradle Wrapper
./gradlew assembleDebug

# 或使用系统Gradle
gradle assembleDebug
```

### 4. 安装测试
```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

## 使用说明

### 首次使用
1. 安装应用后，打开应用主界面
2. 点击"无障碍服务"开关
3. 在系统设置中找到"广告检测工具"
4. 开启无障碍服务权限
5. 返回应用，授予悬浮窗权限

### 功能使用
- **开始检测**：开启无障碍服务后自动开始
- **查看统计**：主界面显示检测计数
- **悬浮窗**：显示实时检测状态
- **震动/声音**：可在主界面或悬浮窗中快速开关

### 权限说明
| 权限 | 用途 |
|------|------|
| BIND_ACCESSIBILITY_SERVICE | 监听屏幕内容 |
| SYSTEM_ALERT_WINDOW | 显示悬浮窗 |
| FOREGROUND_SERVICE | 保持后台运行 |
| VIBRATE | 震动反馈 |
| RECEIVE_BOOT_COMPLETED | 开机自启 |

## 技术实现

### AccessibilityService
- 监听 TYPE_WINDOW_CONTENT_CHANGED、TYPE_WINDOW_STATE_CHANGED 事件
- 遍历节点树，检测符合特征的关闭按钮
- 支持文本匹配和控件ID匹配

### 悬浮窗实现
- 使用 WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY
- 支持拖动定位，自动保存位置
- 可在运行时动态显示/隐藏

### 前台服务
- 使用 FOREGROUND_SERVICE_TYPE_SPECIAL_USE
- 显示持续通知，保持服务存活
- 支持从通知快速操作

## 常见问题

### Q: 为什么需要无障碍权限？
A: Android系统要求使用AccessibilityService才能访问其他应用的内容，检测广告关闭按钮必须在获得此权限后才能实现。

### Q: 会不会影响手机性能？
A: 检测服务只在必要时处理事件，不会持续消耗资源。服务使用 EventTypes 过滤，只在必要时触发。

### Q: 支持自动点击关闭按钮吗？
A: 当前版本仅提供检测和提醒功能，不自动点击，避免误触重要按钮。

### Q: 为什么需要悬浮窗权限？
A: 悬浮窗用于显示检测状态和提供快捷控制，是独立于无障碍服务的高级功能。
