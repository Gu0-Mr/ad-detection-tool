# 广告检测逻辑测试指南

## 更新概述

### 检测逻辑改进
- **旧逻辑**：检测普通关闭按钮（X、关闭、skip等）
- **新逻辑**：检测广告看完后的"可领取奖励"类倒计时结束提示

### 新增两种检测状态

| 状态 | 名称 | 检测目标 | 用户提示 |
|------|------|----------|----------|
| 状态1 | 倒计时进行中 | "5秒后可领取奖励"等 | "广告倒计时中，请等待..." |
| 状态2 | 倒计时结束 | "可领取奖励"+"X关闭按钮" | "可领取奖励！"（强烈提醒） |

## 测试方法

### 方法一：Logcat日志查看

1. 打开Android Studio或ADB工具
2. 设置日志过滤器：`AdDetectorSimple`
3. 运行APP并触发广告
4. 观察日志输出

**预期日志**：
```
# 状态2检测成功
D/AdDetectorSimple: 【检测成功-状态2】检测到倒计时结束，可领取奖励！matchedText=可领取奖励

# 状态1检测成功  
D/AdDetectorSimple: 【检测成功-状态1】检测到广告倒计时中，matchedText=5秒后可领取奖励

# 未检测到
D/AdDetectorSimple: 【检测结果】未检测到广告相关元素
```

### 方法二：断点调试

1. 在`AdDetectorSimple.kt`设置断点：
   - `detectState2Ready()` 方法入口
   - `detectState1Countdown()` 方法入口
   - `callback?.onAdDetected()` 调用处

2. 触发广告后观察：
   - `matchedText` 的具体值
   - `state` 的状态类型

### 方法三：单元测试

创建测试用例验证关键词匹配：

```kotlin
@Test
fun testState2KeywordMatching() {
    val testCases = listOf(
        "可领取奖励" to true,
        "点击领取" to true,
        "立即领取" to true,
        "普通关闭按钮" to false,
        "跳过广告" to false
    )
    
    testCases.forEach { (text, expected) ->
        val matched = REWARD_READY_KEYWORDS.any { text.contains(it) }
        assertEquals(expected, matched)
    }
}

@Test
fun testCountdownPatternMatching() {
    val testCases = listOf(
        "5秒后可领取奖励" to true,
        "3秒后可关闭" to true,
        "倒计时5秒" to false,  // 缺少"后"
        "可领取奖励" to false   // 缺少倒计时
    )
    
    testCases.forEach { (text, expected) ->
        val hasCountdown = COUNTDOWN_NUMBER_KEYWORDS.any { text.contains(it) }
        val hasContext = text.contains("后")
        assertEquals(expected, hasCountdown && hasContext)
    }
}
```

## 常见APP广告样式测试

### 测试APP列表

| APP类型 | 测试APP | 广告样式 |
|---------|---------|----------|
| 短视频 | 抖音/快手 | "5秒后可关闭"+关闭按钮 |
| 资讯 | 今日头条 | "点击领取"+奖励按钮 |
| 游戏 | 各大手游 | "可领取奖励"+X关闭 |
| 电商 | 淘宝/京东 | "跳过广告"+倒计时 |

### 测试步骤

1. **准备测试环境**
   ```bash
   # 确保无障碍服务已开启
   adb shell settings get accessibility_enabled
   # 应返回 1
   ```

2. **启动检测服务**
   ```bash
   # 通过广播启动
   adb shell am broadcast -a com.accessibility.adx.ACTION_START
   ```

3. **触发广告**
   - 打开测试APP
   - 观看视频广告（等待15-60秒）
   - 观察右上角出现"X秒后可..."提示

4. **验证结果**
   - 查看Logcat日志
   - 检查手机是否震动/播放提示音
   - 确认悬浮窗或通知显示

## 故障排查

### 问题1：状态2检测不到

**可能原因**：
- 广告UI结构不是按钮类型
- 关闭按钮和奖励文字不在同一层级

**解决方案**：
- 检查日志中的`matchedText`
- 确认APP广告UI结构

### 问题2：状态1频繁触发

**可能原因**：
- 某些APP的"跳过"按钮也包含"X秒"

**解决方案**：
- 添加更多上下文判断
- 优化关键词组合

### 问题3：位置检测偏差

**可能原因**：
- 不同手机的屏幕密度不同

**解决方案**：
- 使用`getRealMetrics()`获取真实屏幕尺寸
- 确认`screenWidth`和`screenHeight`正确
