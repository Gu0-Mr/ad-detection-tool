## 检测优先级

| 优先级 | 检测内容 | 关键词 | 动作 |
|--------|----------|--------|------|
| **1** | 弹窗检测 | 继续观看、继续领取、再领一次 | 点击弹窗 |
| **2** | 领取成功+关闭 | 领取成功、已领取、恭喜获得 | 点击关闭 |
| **3** | 领取按钮 | 可领取奖励、点击领取、立即领取 | 点击领取 |
| **4** | 倒计时 | X秒后可领取、X秒后可关闭 | 等待 |

## 状态机说明

| 状态 | 说明 | 可能的后续状态 |
|------|------|----------------|
| `IDLE` | 空闲监听中 | COUNTDOWN |
| `COUNTDOWN` | 倒计时进行中 | REWARD_READY, IDLE |
| `REWARD_READY` | 可领取奖励 | CLICKING_REWARD |
| `CLICKING_REWARD` | 正在点击领取 | REWARD_SUCCESS, POPUP_DETECTED |
| `REWARD_SUCCESS` | 领取成功 | CLOSING_AD |
| `CLOSING_AD` | 正在关闭广告 | POPUP_DETECTED, TASK_COMPLETE, IDLE |
| `POPUP_DETECTED` | 检测到弹窗 | CLICKING_REWARD, TASK_COMPLETE |
| `TASK_COMPLETE` | 任务完成 | IDLE |

---

## 防重复机制

### 点击冷却
- 两次点击间隔：3秒（`CLICK_COOLDOWN`）
- 防止对同一按钮重复点击

### 循环限制
- 最大循环次数：10次（`MAX_LOOP_COUNT`）
- 超过限制强制结束任务，防止死循环

### 状态标记
- `hasClickedReward`：已点击过领取，防止重复
- `hasClosedAd`：已关闭过广告，防止重复

## 技术要点

### 点击执行
```kotlin
node.performAction(AccessibilityNodeInfo.ACTION_CLICK)
```

### 状态管理
- 状态机驱动：根据当前状态决定检测优先级
- 回调通知：通过`DetectionCallback`传递状态变化

### 检测区域
```
右上角区域: 右侧85%-100% × 上方0%-25%
```
